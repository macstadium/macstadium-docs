Orka for DevOps is available in two license tiers: Basic and Advanced.

## What's included in each tier?

| Feature | Basic | Advanced |
|---|---|---|
| Node orchestration with priority and grouping | ✓ | ✓ |
| OCI-compatible images | ✓ | ✓ |
| Kubernetes-native interfaces (kubectl and compatible tools) | ✓ | ✓ |
| CI/CD integrations (GitHub Actions, Jenkins, Buildkite, TeamCity) | ✓ | ✓ |
| Packer support for image creation | ✓ | ✓ |
| [Image caching](/orka/orka-resources/image-caching) | — | ✓ |
| Network and VM isolation | — | ✓ |
| Linux workload support | — | ✓ |

## Which tier do I need?

**Basic** covers the typical CI/CD use case: deploy macOS VMs on demand, run builds, save images. If your team is running iOS/macOS build pipelines with GitHub Actions, Jenkins, Buildkite, or TeamCity, Basic is sufficient.

**Advanced** is worth it if you need any of the following:

- **Image caching** — pre-downloads images to nodes before deployment, eliminating pull delays during CI runs. Meaningful for high-throughput pipelines where VM startup time matters.
- **Network and VM isolation** — enforce hard network boundaries between namespaces. Relevant for multi-tenant environments or compliance requirements.
- **Linux workload support** — run Linux containers or workloads alongside your macOS VMs on the same Apple Silicon infrastructure.

## How do I check which license I have?

Contact your MacStadium account team or email [support@macstadium.com](mailto:support@macstadium.com). They can confirm your current tier and what's enabled on your cluster.

## Can I upgrade from Basic to Advanced?

Yes. Email [support@macstadium.com](mailto:support@macstadium.com) to discuss upgrading your license.

## Does the license apply per node or per cluster?

The license applies at the cluster level — all nodes in your Orka cluster operate under the same tier.
